{% comment %} ============================================================ DROPDOWN — composant à propriétés (combobox + listbox ARIA) ============================================================ Rend le déclencheur et le menu. Le CONTENEUR positionné est à la charge de l'appelant (`.dropdown` pour le filtre, `.contact-field` pour le formulaire), car ce dernier contient parfois d'autres éléments (label, champ caché). Le comportement est piloté par un contrôleur unique dans script.js, qui trouve les dropdowns via `[data-dropdown-trigger]` et retrouve leur menu via `aria-controls`. Aucune dépendance aux classes CSS : les deux habillages (`filter` et `form`) partagent le même contrat. Propriétés : id (requis) préfixe des identifiants ("-btn", "-list") options (requis) tableau de { value, label: {fr,en}, icon? } lang (requis) "fr" | "en" variant "filter" (défaut) | "form" label texte de l'aria-label du déclencheur labelledby id d'un élément servant d'étiquette (prioritaire) selected `value` de l'option sélectionnée au chargement placeholder texte affiché tant que rien n'est choisi (variant form) extra tableau de valeurs brutes à ajouter en fin de liste (catégories déclarées nulle part : filet de sécurité) L'état ouvert/fermé est porté par `aria-expanded` sur le déclencheur, et le CSS s'y accroche.
